# Canonical Example｜表达、自我介绍与模拟面试

## 45秒自我介绍

您好，我是一名AI产品方向的在读研究生，过去的核心经历集中在智能座舱语音Agent和模型评测。我参与过主动导航场景设计，并基于真实数据、历史bad case和PRD样本构建了1000多条测试集，设计任务成功、推理规划和交互质量三维评估规则。我最能贡献的是把用户问题转成可评测的数据和产品规则，再协同研发完成内部验证；如果您感兴趣，我可以重点展开测试集或评估器的设计。

## 关键词卡

`AI产品` / `语音Agent` / `1000+测试集` / `三维评估` / `六类问题` / `个人边界` / `内部回归`

## 练习动作

- 录制一次45秒自我介绍，删除简历流水账。
- 将E1-B4控制在90秒，必须讲出一个评分取舍。
- 对“你没有标注平台经验怎么办”练习诚实边界和迁移思路。

## 模拟面试起始题

> 你为什么把评估体系拆成任务成功、推理规划和交互质量三个维度？如果只能保留一个，你选哪个？

运行时不要提前给出标准答案。等待用户回答后，继续追问：

1. 这个选择如何反映业务目标？
2. 有没有出现任务完成但交互质量不合格的案例？
3. 你的评分规则如何降低主观差异？

## 诚实边界

**问：你搭过标注平台吗？**

> 我没有完整搭建过标注平台。我的直接经验是评测集的来源分层、场景标签、问题回流和质量规则；如果进入标注链路，我会先把任务定义、标签体系、一致性检查、抽检和回流机制补齐，但不会把这部分说成已经做过。

**问：模型优化了多少？**

> 项目仍在内部测试，我没有可以对外确认的统一版本指标。我能确认的是意图识别和上下文继承的问题在迭代中逐步减少，以及测试集和评估规则仍被持续使用；我不想把这个趋势包装成精确提升数字。
