# Canonical Example｜输入体检与岗位调研

> 本文件为虚构示例，用于演示 offerpilot_skill 的结构和证据链。人物、公司与数据均不对应真实候选人；不得复制到用户简历。

## 用户输入摘要

- 目标市场：中国大陆。
- 目标岗位：某语音大模型公司的模型数据策略产品实习生。
- 投递状态：尚未投递，可以修改简历。
- 用户材料：完整JD、一页简历、一段智能座舱语音Agent经历。
- 已确认硬条件：上海、每周4天、连续6个月。

## 输入健康

| 字段 | 状态 | 处理 |
| --- | --- | --- |
| 公司/岗位/方向 | Complete | 进入定向流程 |
| 市场与语言 | Complete | 使用中国大陆招聘与面经来源 |
| JD | Complete | 以用户JD为最高权重 |
| 简历 | Complete | 解析并建立证据映射 |
| 投递状态 | Complete | 允许修改 |
| 面试轮次 | Workable | 尚未收到邀请，不生成固定轮次文件 |

结论：**Full模式（简历阶段）+ Interview-lite（轮次未知）**。

## 3-5项核心能力

| 核心能力 | JD依据 | 为什么重要 | 简历需要证明 |
| --- | --- | --- | --- |
| 模型评测体系 | 从0到1定义语音模型评测维度 | 决定版本验收与优化方向 | 测试集、指标、评分规则和评估器 |
| 数据集与质量策略 | 规划评测集/训练集、标注和质量监控 | 数据质量直接影响训练与评测 | 样本来源、分层、回流和质量规则 |
| Bad case闭环 | 回流、归因、修复、回测 | 把问题转成模型迭代输入 | 问题分类、个人边界和回归机制 |
| 跨团队落地 | 与算法、数据、工程搭工具链 | 产品需要推动复杂协作 | 规则、验收、协作对象和交付物 |
| 语音/AI产品sense | 音频理解、生成、全双工和Voice Agent | 需要理解能力边界与体验 | 真实语音交互设计和模型评测经验 |

## 来源覆盖

| 等级 | 示例来源 | 用途 |
| --- | --- | --- |
| A | 用户提供JD、公司招聘页、官方模型文档 | 岗位职责、产品能力和硬条件 |
| B | 同公司/同类岗位近6个月候选人一手面经 | 深挖重点和轮次线索 |
| C | 语音模型行业报告、主流媒体 | 行业和竞品背景 |
| D | SEO求职文章 | 仅作线索，不支撑结论 |

- 示例未写入真实链接，因为目标公司为虚构；真实执行必须给出可点击来源。
- 面试轮次没有可靠证据，保持“未知”，不生成一面/二面/HR面文件。

## 对下游的约束

- 简历必须优先呈现语音Agent评测，而非普通原型或培训工作。
- “改善”只能写内部测试趋势，不写成已上线指标提升。
- 若训练集/标注平台没有真实经历，必须标为能力缺口，不硬塞关键词。
