## 1. 准备边界与轮次地图

先确认：实际投递版本、目标岗位、面试时间、邀请信息和已知轮次。简历已投时冻结文本，不能把之后修改的版本当成面试官手中的版本。

轮次依据优先级：

1. 用户收到的邀请、HR/招聘方信息。
2. 用户提供的目标岗位面经。
3. 同公司同岗位近6个月的多个一手面经。
4. 公司招聘流程说明。
5. 同类岗位经验仅能作为低置信线索。

轮次标记：已确认、高置信推断、待确认。只为前两类生成独立准备；待确认轮次只在总览提醒，除非用户明确要求预备。

```markdown
| 轮次 | 依据 | 置信度 | 可能面试官 | 考察重点 | 是否单独准备 |
```

## 2. Bullet inventory

把实际投递/最终简历逐条编号，例如：

`E1-B1 实习1 / 项目名 / 评测体系`

每个bullet必须覆盖：

| 字段 | 要求 |
| --- | --- |
| 原文 | 与面试官手中版本一致 |
| 对应能力 | 证明JD哪项能力 |
| 30秒讲法 | 背景 -> 个人动作 -> 结果/反思 |
| 60-90秒展开 | 王牌或高风险bullet必备，补决策、取舍和证据 |
| 追问 | 至少2个高概率问题 |
| 二三层追问 | 至少1个继续深挖问题 |
| 证据口径 | 数字、样本、基线、时间、产物、个人贡献 |
| 证据状态 | 能讲清 / 部分记不清 / 不可硬讲 |
| 禁区 | 不能使用的夸大措辞或未做细节 |
| 处理 | 放大 / 正常讲 / 降级 / 转场 / 未来版本回改 |

最终简历任一bullet未覆盖，面试准备不得标记完成。

## 3. 项目讲法与自我介绍

### 项目讲法

对核心项目按以下结构准备自然口语，不要求机械背稿：

1. 背景：谁遇到什么问题，为什么值得做。
2. 目标：成功标准、指标或验收条件。
3. 角色：个人负责什么，别人负责什么。
4. 决策：关键洞察、至少一个真实取舍及原因。
5. 执行：方法、工具、时间线、协作与阻力。
6. 结果：数据口径、交付物、反馈或阶段性变化。
7. 反思：如果重做会改变什么，如何迁移到目标岗位。

技术、AI和业务知识点必须贴回用户自己的项目。例如AI评测要准备样本设计、评分口径、偏差、归因和回归，而不是只背概念定义。

### 自我介绍

输出：

- 30-60秒口语版：身份 -> 最相关经历 -> 岗位匹配 -> 可展开入口。
- 5-7个关键词卡：帮助脱稿，不提供必须逐字背诵的长稿。
- 避雷：不读简历、不一次倒完所有项目、不道歉式开场、不夸大。
- 练习动作：录音/录屏，检查语速、停顿、眼神和是否能在90秒内讲清王牌项目。

## 4. 风险地图与诚实兜底

逐条审计数字、强动词、职责、技术细节和结果归因：

- **可放大**：真实、能讲过程、有证据。
- **正常讲**：真实但不是核心亮点。
- **降级**：事实存在但口径或贡献边界不完整。
- **不可硬讲**：用户无法解释、明显夸大或不属于个人成果。

风险地图：

```markdown
| Claim/Bullet | 风险 | 缺口 | 面试处理 | 未来简历处理 |
```

安全边界句应承认范围并说明验证思路，例如：

> 这部分我负责的是产品规则和验收，底层实现由研发完成；我能具体说明的是规则如何定义、如何验证以及我们发现了哪些问题。

> 这个数字的精确口径我目前无法确认，所以不想给出一个可能误导的答案；我可以讲清当时使用的数据范围、判断方法和阶段性结论。

不要教用户用转移话题掩盖虚假内容。对明确虚假claim，未投版本直接删除；已投版本准备简短纠正和事实边界。

## 5. 轮次定向准备

### 简历/业务初面

- 真实性、个人贡献、数据口径、项目细节。
- 每个核心bullet的30秒讲法和二三层追问。
- 为什么岗位、为什么公司、到岗条件。

### 专业/主管面

- 业务理解、用户与指标、方案取舍、竞品、跨团队推进。
- 把岗位知识点贴回项目，说明决策而非背模板。
- 准备一个失败/冲突/改变方案的真实案例。

### 综合/终面

- 潜力、复杂决策、复盘、长期动机和优先级判断。
- 选择最能体现成长和取舍的项目，不重复流水账。

### HR/recruiter

只有轮次已确认或高置信时单独准备。覆盖动机、稳定性、到岗、地点、周期、薪资/签证和背景风险，不虚构offer或时间安排。

每个轮次准备2-3个有信息量的反问，优先询问团队目标、岗位成功标准、当前关键问题和协作方式；避免询问公开可查的基础信息。

## 6. 模拟面试与面后复盘

### 模拟面试

1. 选择目标轮次和最可能考察点。
2. 一次只问一个问题，等待用户回答。
3. 根据回答追问事实、决策、数据或反思，通常连续2-3层。
4. 回合结束后评价内容、结构、证据和表达，不在提问前泄露标准答案。
5. 把新事实同步回bullet讲法；未经证实内容不得进入简历。

### 面后复盘

记录：公司、岗位、轮次、日期、面试官类型、原题、当场回答、失分原因、下次关键词、需补材料、是否影响简历。

将问题区分为：准备缺口、表达问题、知识缺口、岗位匹配和面试官风格。同一问题再次出现前必须完成针对性练习。

## 7. 文件交付结构

```text
面试准备/
  00-总览.md
  01-简历bullet逐条深挖.md
  02-表达与自我介绍.md
  03-{已确认或高置信轮次}.md
  99-面后复盘.md
```

`00-总览.md` 至少包含：只能记3条、轮次地图、风险地图、bullet覆盖状态和最后准备优先级。

`01` 覆盖全部最终bullet；`02`包含口语自我介绍、关键词卡、核心项目讲法、练习动作和诚实边界；`03+`引用bullet ID，不复制整份长文；`99`提供可持续回填的复盘表。
