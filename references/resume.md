# 简历工作流

## 1. 输入体检

支持对话文本、Markdown、CSV、xlsx、DOCX 和 PDF。无论格式如何，都先识别：

- 基础信息：姓名、电话或邮箱；实习岗位另检查求职意向、每周天数和连续月数。
- 教育经历：学校、专业、学历、起止时间。
- 实习/工作经历：公司、岗位、时间、背景、个人事项；用户没有时可省略。
- 项目经历：项目名称、时间、项目背景、个人事项。
- 个人技能：有经历证据的工具、方法和能力。
- 自我评价：2–3 项由真实经历支撑的岗位优势。

缺项时先从已确认材料可靠提炼；不能确认的内容每轮询问 2–4 项，并给出可直接填写的字段格式。必要信息未补齐或用户未确认模块确实不存在时，不生成终稿。

DOCX 必须递归读取嵌套表格，并用完整文本或 XML 交叉核对教育、求职意向和实习时间。CSV/xlsx 先用 `scripts/csv_to_md.py` 转为内部骨架，再重新执行事实和完整性检查。

## 2. 证据与岗位定位

从 JD 收敛 3–5 项核心能力，建立：

`岗位能力 → 经历 → 个人动作 → 方法/工具 → 交付物/结果 → 证据状态`

- 可使用：用户确认，角色、过程、结果可解释。
- 待确认：事实基本成立，但缺口径、范围、归因或个人边界。
- 不可使用：模型推断、无法解释或团队结果不能归因。

姓名、学校、学历、专业、公司、职位、日期、项目名、奖项和证书属于固定事实，不擅自修改。

## 3. 结构、排序与篇幅

- 用户提供且涉及学校的教育经历全部保留，有几段写几段，不删除或合并学校/学历阶段。
- 同一模块按结束时间倒序；结束时间相同则按开始时间倒序。时间不完整时保留原顺序并询问。
- 校招突出教育、实习、项目、竞赛和潜力；社招突出工作成果、职责深度和业务影响。
- 默认一页：通常为 2–3 段工作/实习 + 1 段项目，或 0–1 段工作/实习 + 2–3 段项目。
- 超量时先询问用户保留策略；不得静默删除教育或重要经历。内容偏少时调整字号、行距和留白，不编造内容。

岗位侧重：

| 方向 | 重点 |
| --- | --- |
| AI Agent / 算法 | 模型或 Agent 方案、评测、数据、指标、工程落地 |
| 后端 | 高并发、稳定性、数据层、性能与成本 |
| 前端 | 工程化、性能体验、组件化、跨端 |
| 产品 | 用户问题、业务目标、取舍、数据验证、推进 |
| 运营 | 拉新/促活/留存、内容/活动策略、漏斗与复盘 |

真实 JD 始终高于岗位预设。

## 4. 经历深挖与改写

每轮只问 2–4 个高价值问题，依次确认：

1. 用户/业务背景、问题、目标和成功标准。
2. 个人 scope、决策权、协作边界和约束。
3. 关键动作、方法、工具、取舍与交付物。
4. 结果、数据口径、时间窗口、基线和个人归因。

每段工作/实习或项目：

- 先写一句项目背景，交代目标用户、核心问题、业务目标和项目定位。
- 通常保留 2–3 件高价值事项；素材不足时宁可保留 1 件完整事项。
- 一条原始事项对应一个加粗小标题和一段正文，不拆分多个标题，也不合并独立事项。
- 小标题只写 4–10 字工作方向，如“场景挖掘”“评测体系”“内容产品迭代”；价值和结果写在正文，不使用“方向｜价值”双段标题。
- 正文内嵌 STAR：场景/目标 → 个人判断与动作 → 方法/协作 → 可归因结果或沉淀。
- 最终 Word 中每条保持 2–3 行；去重复但不删除关键动作、方法和结果。

数字需核对定义、样本、时间、基线、来源和归因。百分比与百分点不得混用。动词强度必须匹配真实角色，不把“参与”机械改成“主导”。

## 5. Word 规范

- 模块：教育背景、可选工作/实习、项目经历、个人技能、自我评价。
- 求职意向和实习时间放在姓名、联系方式下方。
- 姓名与基础信息视觉居中；用户照片固定右上角，高 2.6cm。未提供时使用明确替代块。
- 全文黑色；中文微软雅黑，英文和数字 Times New Roman。
- 模块标题使用 1 磅黑色下边框，不使用色块。
- 所有表格使用“根据窗口调整表格”，宽度占页面可用区域 100%，列宽按比例分配，日期右对齐。
- 生成后渲染检查页数、字体、照片、边框、溢出、留白和每条 2–3 行。

## 6. 自检

- 固定事实是否未改动。
- 每个 JD 关键词是否有经历证据。
- 每个数字、强动词和结果是否可解释。
- 是否错误归因团队成绩。
- 教育经历是否完整。
- 项目背景、小标题和正文结构是否齐全。
- 是否存在重复、占位符、内部标签或过程说明。
- 每个 bullet 是否能连续回答两层追问。
